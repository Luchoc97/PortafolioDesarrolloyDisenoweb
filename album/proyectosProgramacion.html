Sistema web de asignación y gestión inteligente de boletas para sorteos

Desarrollo y funcionamiento del aplicativo

Aplicación web desarrollada en Flask (Python) diseñada para automatizar el proceso de asignación de boletas y gestión de sorteos en campañas promocionales, eliminando procesos manuales y reduciendo errores operativos. El sistema calcula automáticamente la cantidad de boletas que recibe cada participante según el monto acumulado y una regla mínima definida por la campaña. Por ejemplo, si la campaña establece que cada $9.000 equivalen a una boleta, un participante con un monto acumulado de $100.000 obtiene sus boletas mediante el cálculo:

100.000 ÷ 9.000 = 11 boletas

Funcionamiento y cargue de información

A partir de esta lógica, la plataforma asigna de manera automática las boletas correspondientes para cada participante. Funcionamiento de la aplicación La plataforma permite importar bases de datos en formato Excel con la información de los participantes de una campaña. Antes del procesamiento, el sistema valida la estructura de los datos y no permite manejar registros duplicados como su numero de identificacion para evitar inconsistencias en la asignación. Una vez cargada la información: Se importan los participantes al sistema. Se asignan automáticamente las boletas correspondientes. Se generan archivos de resultados para trazabilidad y validación. Se realizan sorteos mediante una interfaz visual dinámica. Se seleccionan ganadores aleatorios respetando las condiciones definidas para cada campaña. Se permite validar participantes elegidos y descartarlos en caso de no cumplir requisitos.
Nota: para el archivo de excel el nombre de las columnas deben ser los siguientes para que el aplicativo pueda leerlos e identificarlos: "agencia" "cedulasociado" "Nombres" "Saldo Total" "NroBoletas"

Caracteristicas adicionales

Una funcionalidad destacada es la capacidad de aplicar reglas especiales, como garantizar que cada oficina o sede tenga al menos un ganador durante el proceso de sorteo. Para brindar mayor transparencia, el sistema incluye una interfaz de revelado tipo "raspa y gana", donde el nombre del ganador permanece oculto hasta completar las verificaciones correspondientes. Además, se desarrolló un módulo adicional para consultar y validar rápidamente la información asociada a una boleta ganadora, mostrando datos como: Número de boleta Nombre del participante Documento de identificación Sede asociada Estado de validación.
